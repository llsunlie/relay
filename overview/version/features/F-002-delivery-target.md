# F-002 交付对象

**里程碑**: v0.0.1 (MVP)

## 用户故事

作为用户，我在设置中添加交付对象（可以是人也可以是系统），后续交付时选择收件对象。

## 入口

系统托盘右键 →「交付对象」

## 添加个人

- 姓名（必填，文本输入）
- 邮箱（必填，文本输入，校验邮箱格式）
- 关系类型（必填，下拉选择：上级、下属、老师、学生、客户、协作伙伴）

## 添加系统

- 系统名称（必填，文本输入）
- 通道地址（必填，文本输入，如 Webhook URL）

## 操作

列表展示已有交付对象，每行显示名称、类型（个人/系统）、关系类型或通道地址，行右侧有编辑和删除按钮。列表顶部有「添加」按钮，点击后先选择类型再填写对应字段。点击交付对象可查看与该对象的交付历史。

## 存储

本地 JSON 文件 `~/.relay/targets.json`。

## 依赖

无

## 成功标准

- 交付对象可增删改查，操作后列表即时刷新
- 数据本地持久化，重启 Agent 后数据不丢失
