# F-001 拖拽交付触发

**里程碑**: v0.0.1 (MVP)

## 用户故事

作为用户，我在系统文件管理器中拖动文件时，系统托盘图标上方浮现一个投放卡片，提示「拖到这里，Relay 交付」，将文件拖入该卡片即触发交付流程。

## 交互流程

1. 用户在 Finder/资源管理器中按下鼠标拖拽文件
2. 系统托盘图标上方浮现投放卡片，文案：「拖到这里，Relay 交付」
3. 用户将文件拖入卡片区域，松手，卡片还会存在，显示刚拖进的文件，出现“取消”和“确定”按钮
4. 点击”确定”，交付窗口出现，文件列表自动填入
5. 用户将文件拖到卡片以外区域松手，卡片消失，不触发交付

## 投放卡片行为

- 拖拽开始时浮现，松手或取消拖拽时消失
- 始终位于系统托盘图标正上方，跟随托盘位置（屏幕右下角 Windows、右上角 macOS）
- 窗口置顶（always-on-top），不被其他窗口遮挡
- 视觉样式：圆角卡片，半透明背景，图标 + 引导文案，支持拖入多个文件
- 文件拖入时有高亮反馈（卡片边框变色）

## 实现要点

- Windows：全局鼠标钩子检测拖拽事件，在 Shell_NotifyIcon 图标位置上方弹出悬浮窗口
- macOS：监听 NSDraggingSession，在 NSStatusItem 位置上方弹出悬浮窗口
- 拖拽数据检测：判断拖拽数据是否包含文件路径，无文件时卡片不浮现
- 投放卡片窗口设为 always-on-top，无任务栏条目

## 依赖

无

## 成功标准

- macOS 和 Windows 上拖拽文件时卡片正确浮现于托盘图标上方
- 投放文件后交付窗口正确弹出，文件列表完整
- 取消拖拽时卡片正确消失，无残留窗口
- 拖拽非文件内容时卡片不出现
