# F-005 邮件投递

**里程碑**: v0.0.1 (MVP)

## 用户故事

作为用户，在交付窗口中点击「确认交付」后，系统自动通过我配置的邮箱 SMTP 将文件作为邮件附件发送给收件人，我不需要再操作邮箱客户端。

## SMTP 配置

在 Agent 设置页面中一次性填写：
- SMTP 服务器地址（如 `smtp.qq.com`）
- 端口（如 `465`）
- 发件人邮箱
- 授权码（非邮箱登录密码）

配置完成后 Agent 测试发送一封验证邮件到发件人邮箱，验证通过后保存配置。

## 邮件格式

- 标题：`[Relay] {交付类型} - {文件名}`
- 正文：留言内容（留言为空时正文仅列出文件列表）
- 附件：原始文件，保持文件名不变

## 发送后行为

系统托盘通知：「已送达 {收件人姓名}」。发送失败时通知：「交付失败：{错误原因}」，交付记录中标记为失败状态。

## 依赖

- F-003 交付窗口

## 成功标准

- 邮件成功发送并出现在收件人邮箱
- 附件和留言内容完整不丢失
- SMTP 配置错误时有明确提示
