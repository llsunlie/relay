# F-009 交付前校验

**里程碑**: v0.0.2 (文件夹监听)

## 用户故事

作为用户，当交付无法正常完成时，系统在交付窗口打开前就通知我问题所在，而不是让我填完信息后才报错。

## 校验项

收件人存在性：规则模板预填的收件人角色在交付对象中是否有对应联系人。

## 校验时机

交付窗口打开前执行。静默模式下在校验通过后才执行静默发送。

## 校验失败行为

托盘通知：「无法交付：未找到{角色}对应的交付对象，请检查交付对象」。不弹出交付窗口。静默模式下不执行发送。

## 依赖

- F-002 交付对象
- F-007 文件夹监听引擎

## 成功标准

- 收件人缺失时必定弹出通知
- 校验失败不阻塞其他正常交付
